## 3.1 SQLite 设计方案

### 3.1.1 为什么选 better-sqlite3

| 决策因素 | better-sqlite3 | drizzle-orm / prisma |
|---------|---------------|---------------------|
| API 模型 | **同步** — 与 SQLite 单线程模型天然匹配 | 异步 — 增加不必要的事件循环开销 |
| 性能 | ~100 万 ops/sec 简单查询 | ~10 万 ops/sec（ORM 抽象开销） |
| 依赖数 | 1 个（native addon） | 多个（schema builder, migrator, etc.） |
| 代码量 | 直接 SQL，简洁明了 | Schema 定义 + 生成代码 |
| 调试 | SQL 即代码，可直接在 sqlite3 CLI 验证 | 需要理解 ORM 生成的 SQL |

**结论**：对于目标 <10,000 行代码的轻量项目，ORM 是负债而非资产。直接使用 better-sqlite3。

### 3.1.2 数据库初始化与 PRAGMA 调优

```typescript
// src/db.ts
import Database from 'better-sqlite3';

export function initDatabase(dbPath: string): Database.Database {
  const db = new Database(dbPath);

  // ---- 性能 PRAGMA ----
  db.pragma('journal_mode = WAL');         // Write-Ahead Logging：读写并发
  db.pragma('synchronous = NORMAL');       // WAL 模式下 NORMAL 足够安全且更快
  db.pragma('cache_size = -20000');        // 20MB 页面缓存（负号 = KB）
  db.pragma('busy_timeout = 5000');        // 锁等待超时 5s
  db.pragma('foreign_keys = ON');          // 启用外键约束

  // ---- 安全 PRAGMA ----
  db.pragma('secure_delete = ON');         // 删除数据时零填充（防残留）

  // ---- 执行迁移 ----
  runMigrations(db);

  return db;
}
```

**WAL 模式要点**：

- 读操作不阻塞写操作，写操作不阻塞读操作
- 单实例场景（VigilClaw）下性能提升显著
- 需要注意 WAL 文件膨胀：定期 checkpoint

```typescript
// 每 5 分钟检查 WAL 文件大小，超过 50MB 时主动 checkpoint
setInterval(() => {
  try {
    const walPath = dbPath + '-wal';
    const stat = fs.statSync(walPath);
    if (stat.size > 50 * 1024 * 1024) {
      db.pragma('wal_checkpoint(RESTART)');
      logger.debug('WAL checkpoint completed');
    }
  } catch {
    // WAL 文件不存在（正常：可能刚 checkpoint 过）
  }
}, 300_000).unref();
```

### 3.1.3 迁移方案

使用 SQLite 内置的 `user_version` PRAGMA 做版本控制，零依赖：

```typescript
interface Migration {
  version: number;
  description: string;
  up: string;
}

const MIGRATIONS: Migration[] = [
  {
    version: 1,
    description: 'Initial schema',
    up: SCHEMA_V1,     // 见 3.2 节完整 SQL
  },
  // 后续版本追加
  // {
  //   version: 2,
  //   description: 'Add memories table for Phase 2',
  //   up: `CREATE TABLE memories (...);`,
  // },
];

function runMigrations(db: Database.Database): void {
  const currentVersion = db.pragma('user_version', { simple: true }) as number;

  const pending = MIGRATIONS.filter(m => m.version > currentVersion);
  if (pending.length === 0) return;

  // 所有迁移在一个事务中执行（原子性）
  const migrate = db.transaction(() => {
    for (const migration of pending) {
      logger.info({ version: migration.version, desc: migration.description }, 'Running migration');
      db.exec(migration.up);
      db.pragma(`user_version = ${migration.version}`);
    }
  });

  migrate();
  logger.info({ from: currentVersion, to: pending.at(-1)!.version }, 'Migrations complete');
}
```

### 索引设计说明

| 索引 | 服务的查询场景 | 预期 QPS |
|------|-------------|---------|
| `idx_messages_session` | 加载会话上下文（最近 N 条） | 每条消息触发 1 次 |
| `idx_api_calls_user_date` | `/cost` 命令按日聚合 | 用户主动查询，低频 |
| `idx_api_calls_task` | 任务完成时汇总费用 | 每个任务 1 次 |
| `idx_tasks_status` | 调度器查找 pending 任务 | 每 30 秒 1 次 |
| `idx_scheduled_next` | 查找到期定时任务 | 每 30 秒 1 次 |
| `idx_security_severity` | 安全事件查询（按严重度） | 极低频，管理员查询 |

## 3.3 数据访问层（DAL）

将所有 SQL 封装为类型安全的方法，避免 SQL 散落在业务代码中：

```typescript
// src/db.ts — 核心查询方法

export class VigilClawDB {
  private db: Database.Database;

  // ---- Prepared Statements（性能关键）----
  private stmts: {
    insertMessage: Database.Statement;
    getRecentMessages: Database.Statement;
    insertApiCall: Database.Statement;
    getUserDayCost: Database.Statement;
    getUserMonthCost: Database.Statement;
    insertTask: Database.Statement;
    updateTaskStatus: Database.Statement;
    getOrCreateUser: Database.Statement;
    upsertCredential: Database.Statement;
    getCredential: Database.Statement;
    insertSecurityEvent: Database.Statement;
    getDueScheduledTasks: Database.Statement;
  };

  constructor(dbPath: string) {
    this.db = initDatabase(dbPath);
    this.stmts = this.prepareStatements();
  }

  private prepareStatements() {
    return {
      insertMessage: this.db.prepare(`
        INSERT INTO messages (session_key, user_id, group_id, role, content, tokens)
        VALUES (?, ?, ?, ?, ?, ?)
      `),

      getRecentMessages: this.db.prepare(`
        SELECT role, content FROM messages
        WHERE session_key = ?
        ORDER BY created_at DESC
        LIMIT ?
      `),

      insertApiCall: this.db.prepare(`
        INSERT INTO api_calls (task_id, user_id, group_id, provider, model, input_tokens, output_tokens, cost_usd)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?)
      `),

      getUserDayCost: this.db.prepare(`
        SELECT COALESCE(SUM(cost_usd), 0) as total
        FROM api_calls
        WHERE user_id = ? AND date(created_at) = date('now')
      `),

      getUserMonthCost: this.db.prepare(`
        SELECT COALESCE(SUM(cost_usd), 0) as total
        FROM api_calls
        WHERE user_id = ? AND strftime('%Y-%m', created_at) = strftime('%Y-%m', 'now')
      `),

      insertTask: this.db.prepare(`
        INSERT INTO tasks (id, user_id, group_id, status, input_summary, created_at)
        VALUES (?, ?, ?, 'pending', ?, datetime('now'))
      `),

      updateTaskStatus: this.db.prepare(`
        UPDATE tasks
        SET status = ?, output_summary = ?, total_cost_usd = ?, error_message = ?,
            started_at = CASE WHEN ? = 'running' THEN datetime('now') ELSE started_at END,
            completed_at = CASE WHEN ? IN ('completed','failed','timeout') THEN datetime('now') ELSE completed_at END
        WHERE id = ?
      `),

      getOrCreateUser: this.db.prepare(`
        INSERT INTO users (id, name) VALUES (?, ?)
        ON CONFLICT(id) DO UPDATE SET name = excluded.name
        RETURNING *
      `),

      upsertCredential: this.db.prepare(`
        INSERT INTO credentials (provider, key_encrypted, iv)
        VALUES (?, ?, ?)
        ON CONFLICT(provider) DO UPDATE SET
          key_encrypted = excluded.key_encrypted,
          iv = excluded.iv,
          last_rotated_at = datetime('now')
      `),

      getCredential: this.db.prepare(`
        SELECT key_encrypted, iv FROM credentials WHERE provider = ?
      `),

      insertSecurityEvent: this.db.prepare(`
        INSERT INTO security_events (event_type, user_id, severity, details)
        VALUES (?, ?, ?, ?)
      `),

      getDueScheduledTasks: this.db.prepare(`
        SELECT * FROM scheduled_tasks
        WHERE enabled = 1 AND next_run_at <= datetime('now')
        ORDER BY next_run_at ASC
      `),
    };
  }

  // ---- 消息相关 ----

  insertMessage(msg: {
    sessionKey: string;
    userId: string;
    groupId?: string;
    role: string;
    content: string;
    tokens?: number;
  }): void {
    this.stmts.insertMessage.run(
      msg.sessionKey, msg.userId, msg.groupId ?? null,
      msg.role, msg.content, msg.tokens ?? null
    );
  }

  getRecentMessages(sessionKey: string, limit: number): Array<{ role: string; content: string }> {
    const rows = this.stmts.getRecentMessages.all(sessionKey, limit) as any[];
    return rows.reverse();  // DESC → 正序
  }

  deleteMessages(sessionKey: string): void {
    this.db.prepare('DELETE FROM messages WHERE session_key = ?').run(sessionKey);
  }

  // ---- 成本相关 ----

  recordApiCall(call: {
    taskId: string;
    userId: string;
    groupId?: string;
    provider: string;
    model: string;
    inputTokens: number;
    outputTokens: number;
    costUsd: number;
  }): void {
    this.stmts.insertApiCall.run(
      call.taskId, call.userId, call.groupId ?? null,
      call.provider, call.model,
      call.inputTokens, call.outputTokens, call.costUsd
    );
  }

  getUserDayCost(userId: string): number {
    return (this.stmts.getUserDayCost.get(userId) as any).total;
  }

  getUserMonthCost(userId: string): number {
    return (this.stmts.getUserMonthCost.get(userId) as any).total;
  }

  /**
   * /cost 命令的完整数据
   */
  getCostReport(userId: string): CostReport {
    const dayCost = this.getUserDayCost(userId);
    const monthCost = this.getUserMonthCost(userId);

    const user = this.db.prepare('SELECT max_cost_per_day, max_cost_per_month FROM users WHERE id = ?')
      .get(userId) as any;

    const modelBreakdown = this.db.prepare(`
      SELECT model, COUNT(*) as call_count, SUM(cost_usd) as total_cost
      FROM api_calls
      WHERE user_id = ? AND date(created_at) = date('now')
      GROUP BY model
      ORDER BY total_cost DESC
    `).all(userId) as any[];

    const topTasks = this.db.prepare(`
      SELECT t.input_summary, t.total_cost_usd
      FROM tasks t
      WHERE t.user_id = ? AND date(t.created_at) = date('now') AND t.total_cost_usd > 0
      ORDER BY t.total_cost_usd DESC
      LIMIT 5
    `).all(userId) as any[];

    return {
      dayCost,
      monthCost,
      dayBudget: user?.max_cost_per_day ?? 10.0,
      monthBudget: user?.max_cost_per_month ?? 100.0,
      modelBreakdown,
      topTasks,
    };
  }

  // ---- 凭证相关 ----

  getCredential(provider: string): { keyEncrypted: Buffer; iv: Buffer } | null {
    const row = this.stmts.getCredential.get(provider) as any;
    if (!row) return null;
    return { keyEncrypted: row.key_encrypted, iv: row.iv };
  }

  upsertCredential(provider: string, keyEncrypted: Buffer, iv: Buffer): void {
    this.stmts.upsertCredential.run(provider, keyEncrypted, iv);
  }

  // ---- 安全事件 ----

  insertSecurityEvent(event: {
    eventType: string;
    userId?: string;
    severity: string;
    details: Record<string, unknown>;
  }): void {
    this.stmts.insertSecurityEvent.run(
      event.eventType, event.userId ?? null,
      event.severity, JSON.stringify(event.details)
    );
  }

  // ---- 清理 ----

  close(): void {
    this.db.close();
  }
}

// ---- 类型定义 ----

export interface CostReport {
  dayCost: number;
  monthCost: number;
  dayBudget: number;
  monthBudget: number;
  modelBreakdown: Array<{ model: string; call_count: number; total_cost: number }>;
  topTasks: Array<{ input_summary: string; total_cost_usd: number }>;
}
```

### Prepared Statement 的性能意义

better-sqlite3 的 Prepared Statement 只编译一次 SQL，后续执行只传参：

| 操作 | 未 prepare | 已 prepare |
|------|-----------|-----------|
| 简单 INSERT | ~50,000 ops/sec | ~200,000 ops/sec |
| 简单 SELECT | ~100,000 ops/sec | ~500,000 ops/sec |

在 VigilClaw 的使用频率下（每秒几十次查询）差异不大，但这是零成本的最佳实践。

## 3.4 成本追踪系统

### 3.4.1 费用计算

```typescript
// src/cost-guard.ts

/**
 * 模型价格表（$/1M tokens）
 * 数据来源：各 Provider 官网
 * 更新策略：代码内硬编码 + 配置文件可覆盖
 */
const MODEL_PRICING: Record<string, { input: number; output: number }> = {
  // Anthropic Claude
  'claude-sonnet-4-5-20250929':   { input: 3.0,   output: 15.0 },
  'claude-haiku-3-5-20250929':    { input: 1.0,   output: 5.0  },
  'claude-opus-4-20250929':       { input: 15.0,  output: 75.0 },

  // Phase 2: OpenAI
  // 'gpt-4o':                    { input: 2.5,   output: 10.0 },
  // 'gpt-4o-mini':               { input: 0.15,  output: 0.6  },
};

export function calculateCost(
  model: string,
  inputTokens: number,
  outputTokens: number,
): number {
  const pricing = MODEL_PRICING[model];
  if (!pricing) {
    logger.warn({ model }, 'Unknown model for cost calculation, using 0');
    return 0;
  }

  return (inputTokens / 1_000_000) * pricing.input
       + (outputTokens / 1_000_000) * pricing.output;
}
```

### 3.4.2 成本记录时机

成本记录发生在 **宿主机** 收到容器 IPC 结果后（而不是容器内部），确保数据由可信进程写入：

```typescript
// src/group-queue.ts — executeTask 中
private async executeTask(task: QueuedTask): Promise<void> {
  const containerRunner = this.containerRunner;

  try {
    // 1. 更新任务状态
    this.db.updateTaskStatus(task.id, 'running');

    // 2. 运行容器
    const result = await containerRunner.runTask(task);

    // 3. ★ 计算并记录成本（宿主机侧，可信数据）
    const cost = calculateCost(
      result.response.model,
      result.response.usage.inputTokens,
      result.response.usage.outputTokens,
    );

    this.db.recordApiCall({
      taskId: task.id,
      userId: task.userId,
      groupId: task.groupId,
      provider: 'anthropic',
      model: result.response.model,
      inputTokens: result.response.usage.inputTokens,
      outputTokens: result.response.usage.outputTokens,
      costUsd: cost,
    });

    // 4. 更新任务完成状态
    this.db.updateTaskStatus(task.id, 'completed', {
      outputSummary: result.response.content.slice(0, 200),
      totalCost: cost,
    });

    // 5. 保存 Assistant 回复到会话历史
    this.sessionManager.saveResponse(task.userId, task.groupId, result.response.content);

    // 6. 回复用户
    await task.replyFn(result.response.content);

  } catch (err) {
    this.db.updateTaskStatus(task.id, 'failed', {
      error: (err as Error).message,
    });
    await task.replyFn(`⚠️ 任务执行失败: ${(err as Error).message}`);
  }
}
```

**注意**：Agent Runner 在容器内也会发起多轮 API 调用（ReAct 循环中可能有多次 `messages.create`），但 token 用量在最终 result 中汇总返回。如果需要精确到每轮调用的记录，可以在 Credential Proxy 层记录（见第二篇 2.4.2 `recordUsage`）。

### 3.4.3 `/cost` 命令输出格式

```typescript
function formatCostReport(report: CostReport): string {
  const dayPercent = Math.round((report.dayCost / report.dayBudget) * 100);
  const monthPercent = Math.round((report.monthCost / report.monthBudget) * 100);

  let output = `📊 **费用报告**\n\n`;
  output += `今日消耗: $${report.dayCost.toFixed(2)} / $${report.dayBudget.toFixed(2)} (${dayPercent}%)\n`;
  output += `本月消耗: $${report.monthCost.toFixed(2)} / $${report.monthBudget.toFixed(2)} (${monthPercent}%)\n`;

  // 模型分解
  if (report.modelBreakdown.length > 0) {
    output += `\n📋 **模型明细 (今日)**\n`;
    for (const item of report.modelBreakdown) {
      const shortName = item.model.replace('claude-', '').replace(/-\d+$/, '');
      output += `• ${shortName}: $${item.total_cost.toFixed(2)} (${item.call_count} 次)\n`;
    }
  }

  // 高消耗任务
  if (report.topTasks.length > 0) {
    output += `\n🔝 **高消耗任务 (今日)**\n`;
    for (const [i, task] of report.topTasks.entries()) {
      const summary = task.input_summary?.slice(0, 30) ?? '(无摘要)';
      output += `${i + 1}. ${summary}... — $${task.total_cost_usd.toFixed(2)}\n`;
    }
  }

  // 预算告警
  if (dayPercent >= 80) {
    output += `\n⚠️ 今日预算已使用 ${dayPercent}%，接近上限！`;
  }

  return output;
```

## 3.5 预算控制系统

### 3.5.1 预算检查流程

```typescript
// src/cost-guard.ts

export interface BudgetStatus {
  exceeded: boolean;
  reason?: 'day_limit' | 'month_limit' | 'task_limit';
  currentCost: number;
  budgetLimit: number;
  remaining: number;
}

export class CostGuard {
  constructor(private db: VigilClawDB) {}

  /**
   * 任务启动前检查预算
   * 在 Router.handleMessage 中调用
   */
  checkBudget(userId: string): BudgetStatus {
    const user = this.db.getUser(userId);
    if (!user) {
      // 新用户，使用默认预算
      return { exceeded: false, currentCost: 0, budgetLimit: 10.0, remaining: 10.0 };
    }

    // 检查日预算
    const dayCost = this.db.getUserDayCost(userId);
    if (dayCost >= user.maxCostPerDay) {
      return {
        exceeded: true,
        reason: 'day_limit',
        currentCost: dayCost,
        budgetLimit: user.maxCostPerDay,
        remaining: 0,
      };
    }

    // 检查月预算
    const monthCost = this.db.getUserMonthCost(userId);
    if (monthCost >= user.maxCostPerMonth) {
      return {
        exceeded: true,
        reason: 'month_limit',
        currentCost: monthCost,
        budgetLimit: user.maxCostPerMonth,
        remaining: 0,
      };
    }

    return {
      exceeded: false,
      currentCost: dayCost,
      budgetLimit: user.maxCostPerDay,
      remaining: user.maxCostPerDay - dayCost,
    };
  }

  /**
   * 格式化预算超限回复
   */
  formatExceededMessage(status: BudgetStatus): string {
    if (status.reason === 'day_limit') {
      return `⛔ 今日预算已用完。\n消耗: $${status.currentCost.toFixed(2)} / $${status.budgetLimit.toFixed(2)}\n预算将于 UTC 00:00 重置。\n\n管理员可使用 /budget <金额> 调整上限。`;
    }
    if (status.reason === 'month_limit') {
      return `⛔ 本月预算已用完。\n消耗: $${status.currentCost.toFixed(2)} / $${status.budgetLimit.toFixed(2)}\n预算将于下月 1 日重置。`;
    }
    return '⛔ 预算已用完。';
  }
}
```

### 3.5.2 预算管理命令

| 命令 | 权限 | 功能 |
|------|------|------|
| `/budget` | 所有用户 | 查看当前预算设置 |
| `/budget <day> [month]` | admin | 设置日/月预算 |
| `/reset-budget [@user]` | admin | 手动重置某用户预算周期 |

```typescript
// 命令处理示例
async function handleBudgetCommand(userId: string, args: string[]): Promise<string> {
  if (args.length === 0) {
    // 查看预算
    const user = db.getUser(userId);
    const dayCost = db.getUserDayCost(userId);
    const monthCost = db.getUserMonthCost(userId);
    return `💰 **预算设置**\n\n` +
      `日预算: $${user.maxCostPerDay.toFixed(2)} (已用 $${dayCost.toFixed(2)})\n` +
      `月预算: $${user.maxCostPerMonth.toFixed(2)} (已用 $${monthCost.toFixed(2)})`;
  }

  // 设置预算（需要 admin 权限）
  if (!isAdmin(userId)) return '⛔ 仅管理员可修改预算。';

  const dayLimit = parseFloat(args[0]);
  const monthLimit = args[1] ? parseFloat(args[1]) : dayLimit * 30;

  if (isNaN(dayLimit) || dayLimit <= 0) return '❌ 无效金额。用法: /budget 20 [600]';

  db.updateUserBudget(userId, dayLimit, monthLimit);
  return `✅ 预算已更新: 日 $${dayLimit.toFixed(2)} / 月 $${monthLimit.toFixed(2)}`;
}
```

### 3.5.3 自动预算重置

预算重置不需要实际清除数据 —— `getUserDayCost()` 查询已经用 `date(created_at) = date('now')` 过滤了当天数据。月预算同理。

这意味着：
- **无需 cron job 做重置** — SQL 查询天然按时间范围聚合
- **历史数据完整保留** — 可以回溯任意时间段的消耗
- **时区处理**：`datetime('now')` 使用 UTC，与配置的 `budgetResetHour` 对齐

如果用户配置了非 UTC 0 点的重置时间（如北京时间 00:00 = UTC 16:00），需要调整 SQL：

```sql
-- 按用户时区计算"今天"的起始时间
-- 假设 reset_hour = 16 (UTC), 则"今天"从 UTC 16:00 开始
SELECT COALESCE(SUM(cost_usd), 0) as total
FROM api_calls
WHERE user_id = ?
  AND created_at >= datetime('now', '-' || (
    CASE WHEN strftime('%H','now') >= '16' THEN 0 ELSE 1 END
  ) || ' days', '+16 hours', 'start of day', '+16 hours')
```

MVP 简化处理：统一使用 UTC 00:00 重置，Phase 2 支持自定义时区。

```typescript
// src/db.ts

export function cleanupOldData(db: Database.Database): void {
  const cleanup = db.transaction(() => {
    // 清理 90 天前的消息
    const msgResult = db.prepare(
      "DELETE FROM messages WHERE created_at < datetime('now', '-90 days')"
    ).run();

    // 清理 365 天前的 API 调用记录
    const apiResult = db.prepare(
      "DELETE FROM api_calls WHERE created_at < datetime('now', '-365 days')"
    ).run();

    // 清理 365 天前的任务记录
    const taskResult = db.prepare(
      "DELETE FROM tasks WHERE created_at < datetime('now', '-365 days')"
    ).run();

    // 清理 365 天前的安全事件
    const secResult = db.prepare(
      "DELETE FROM security_events WHERE created_at < datetime('now', '-365 days')"
    ).run();

    logger.info({
      messages: msgResult.changes,
      apiCalls: apiResult.changes,
      tasks: taskResult.changes,
      securityEvents: secResult.changes,
    }, 'Data cleanup completed');
  });

  cleanup();

  // 清理后执行 VACUUM 回收空间
  db.exec('VACUUM');
}
```
