## 1.1 架构总览

### 1.1.1 设计哲学与约束

VigilClaw 的架构基于一个核心假设：**AI Agent 是不可信的**。

无论多强的提示词工程，都无法从根本上防止 Prompt Injection。因此安全边界不能放在应用层（OpenClaw 的 allowlist/denylist），而必须下沉到**操作系统内核层**（容器隔离 + 网络命名空间）。

这一假设直接决定了双进程架构的选择：

| 设计决策 | 原则 | 落地 |
|---------|------|------|
| Host + Container 分离 | 不信任 Agent | Agent 运行在隔离容器中，即使被攻陷也无法影响宿主机 |
| Anthropic SDK 直接调用 | 可审计 | 不依赖闭源的 Claude Agent SDK，所有代码可审计 |
| 文件系统 IPC | 稳定优先 | 容器挂载目录天然支持，无需额外网络配置 |
| Telegram 优先 | 稳定优先 | 官方 Bot API，无逆向工程风险 |
| SQLite 单库 | 轻量 | 零外部依赖，事务一致性，单机场景足够 |

**硬约束清单：**

| 约束 | 目标值 | 测量方法 |
|------|--------|---------|
| 核心代码量 | <5,000 行 TS (MVP) / <10,000 行 (Phase 2) | `cloc src/` |
| 生产依赖数 | <50 npm 包（目标 ~8 个） | `pnpm list --depth=0` |
| 容器启动时间 | <3s | 从 IPC 请求到容器就绪 |
| 消息首字节延迟 | <2s | 从收到 Telegram 消息到开始回复 |
| 宿主机内存 | <100MB RSS | 空闲状态 |

### 1.1.3 进程职责划分

#### 宿主机进程 (Host Process)

单个 Node.js 长驻进程，负责**一切非推理**工作。

| 做 | 不做 |
|---|---|
| 接收和发送 Telegram 消息 | ❌ 永远不运行 AI 推理 |
| 创建/监控/销毁 Docker 容器 | ❌ 永远不执行用户的 Bash 命令 |
| 管理 SQLite 数据库（消息、成本、配置） | ❌ 永远不直接调用 LLM API |
| 通过 Credential Proxy 注入 API Key | ❌ 永远不把 API Key 传入容器环境变量 |
| 执行速率限制和预算检查 | ❌ 永远不信任容器内返回的费用数据 |
| 监控 IPC 目录获取 Agent 输出 | ❌ 永远不解析或修改 Agent 的推理过程 |
| 调度定时任务、管理延迟队列 | ❌ 永远不直接与外部 API 通信（除 Telegram） |

#### 容器进程 (Container Process)

每次任务启动一个临时 Docker 容器，用后即毁。

| 做 | 不做 |
|---|---|
| 运行 Agent ReAct 循环 | ❌ 不存储任何持久状态 |
| 调用 Anthropic API（通过 Credential Proxy） | ❌ 不直接持有 API Key |
| 执行工具（Bash/Read/Write/Edit） | ❌ 不访问宿主机文件系统（除挂载目录） |
| 将结果写入 IPC 输出目录 | ❌ 不访问内网或非白名单外网 |
| 轮询 IPC 输入目录接收新消息 | ❌ 不修改自身的网络/安全配置 |

## 1.2 核心模块详细设计

### 1.2.1 入口与编排器 (`src/index.ts`)

**职责**：系统启动入口，负责模块初始化和生命周期管理。

#### 初始化流程

```typescript
async function main() {
  // 1. 加载配置（环境变量 > 配置文件 > 默认值）
  const config = loadConfig();

  // 2. 初始化 SQLite（创建表 + 执行迁移）
  const db = initDatabase(config.dbPath);

  // 3. 启动 Credential Proxy（HTTP 代理服务器）
  const credentialProxy = new CredentialProxy(db, config.masterKey);
  await credentialProxy.start();

  // 4. 初始化核心模块
  const costGuard = new CostGuard(db);
  const sessionManager = new SessionManager(db, config.contextLength);
  const containerRunner = new ContainerRunner(config.docker, credentialProxy);
  const groupQueue = new GroupQueue(config.maxConcurrentContainers);
  const taskScheduler = new TaskScheduler(db, groupQueue);
  const router = new Router(db, costGuard, sessionManager, groupQueue);

  // 5. 注册并启动渠道
  const channels: IChannel[] = [];
  if (config.telegram.enabled) {
    channels.push(new TelegramChannel(config.telegram, router));
  }
  for (const channel of channels) {
    await channel.start();
  }

  // 6. 启动任务调度器
  taskScheduler.start();

  // 7. 注册优雅关闭
  setupGracefulShutdown(channels, containerRunner, credentialProxy, db);

  logger.info('VigilClaw started');
}
```

**初始化顺序约束**（必须严格遵守）：

```
Config → DB → CredentialProxy → CostGuard → SessionManager
  → ContainerRunner → GroupQueue → TaskScheduler → Router → Channels
```

理由：每一层依赖前一层。Channel 必须最后启动，因为一旦启动就可能收到消息。

#### 优雅关闭

```typescript
function setupGracefulShutdown(
  channels: IChannel[],
  containerRunner: ContainerRunner,
  credentialProxy: CredentialProxy,
  db: Database
) {
  const shutdown = async (signal: string) => {
    logger.info({ signal }, 'Shutting down...');

    // 1. 停止接收新消息
    for (const channel of channels) {
      await channel.stop();
    }

    // 2. 等待正在运行的容器完成（最多 30s）
    await containerRunner.drainAll(30_000);

    // 3. 停止 Credential Proxy
    await credentialProxy.stop();

    // 4. 关闭数据库连接
    db.close();

    process.exit(0);
  };

  process.on('SIGINT', () => shutdown('SIGINT'));
  process.on('SIGTERM', () => shutdown('SIGTERM'));
}
```

### 1.2.3 消息路由器 (`src/router.ts`)

**职责**：接收渠道消息，经过安全检查后创建任务。

#### 核心流程

```typescript
export class Router {
  constructor(
    private db: Database,
    private costGuard: CostGuard,
    private sessionManager: SessionManager,
    private groupQueue: GroupQueue,
  ) {}

  async handleMessage(msg: IncomingMessage): Promise<void> {
    // 1. 速率限制检查
    if (this.isRateLimited(msg.userId, msg.groupId)) {
      return this.replyRateLimited(msg);
    }

    // 2. 预算检查
    const budgetStatus = this.costGuard.checkBudget(msg.userId);
    if (budgetStatus.exceeded) {
      return this.replyBudgetExceeded(msg, budgetStatus);
    }

    // 3. 命令解析（/cost, /model, /clear 等）
    if (msg.text?.startsWith('/')) {
      return this.handleCommand(msg);
    }

    // 4. 加载/创建会话上下文
    const context = this.sessionManager.getContext(msg.userId, msg.groupId);

    // 5. 存储消息到 SQLite
    this.db.insertMessage({
      userId: msg.userId,
      groupId: msg.groupId,
      role: 'user',
      content: msg.text ?? '',
      timestamp: msg.timestamp,
    });

    // 6. 创建任务并入队
    const task = this.createTask(msg, context);
    this.groupQueue.enqueue(task);
  }
}
```

#### 命令处理

| 命令 | 功能 | 示例 |
|------|------|------|
| `/cost` | 查看当前消耗 | 今日 $3.24 / $10.00 |
| `/model <name>` | 切换模型 | `/model haiku` |
| `/clear` | 清空上下文 | 重置会话历史 |
| `/budget <amount>` | 设置日预算 | `/budget 20` |
| `/help` | 帮助信息 | 命令列表 |

### 1.2.4 会话管理器 (`src/session-manager.ts`)

**职责**：管理用户/群组的对话上下文。

```typescript
export class SessionManager {
  constructor(
    private db: Database,
    private contextLength: number,  // 默认 20
  ) {}

  /**
   * 获取会话上下文（最近 N 条消息）
   */
  getContext(userId: string, groupId?: string): Message[] {
    const sessionKey = groupId ?? userId;
    return this.db.getRecentMessages(sessionKey, this.contextLength);
  }

  /**
   * 清空会话上下文
   */
  clearContext(userId: string, groupId?: string): void {
    const sessionKey = groupId ?? userId;
    this.db.deleteMessages(sessionKey);
  }

  /**
   * 保存 Agent 回复到上下文
   */
  saveResponse(userId: string, groupId: string | undefined, content: string): void {
    this.db.insertMessage({
      userId,
      groupId,
      role: 'assistant',
      content,
      timestamp: new Date(),
    });
  }
}
```

**设计要点**：

- 每个 `userId + groupId` 组合有独立上下文
- 私聊：`sessionKey = userId`
- 群组：`sessionKey = groupId`（群组内所有人共享上下文）
- 超过 `contextLength` 时自动截断旧消息（简单丢弃，Phase 2 升级为 LLM 摘要压缩）

### 1.2.5 任务队列 (`src/group-queue.ts`)

**职责**：群组级任务排队和并发控制。

```typescript
interface QueuedTask {
  id: string;                    // UUID
  userId: string;
  groupId?: string;
  messages: Message[];           // 会话上下文
  model: string;
  tools: string[];
  createdAt: Date;
  replyFn: (text: string) => Promise<void>;  // 回复回调
}

export class GroupQueue {
  private queues = new Map<string, QueuedTask[]>();  // groupId → tasks
  private activeCount = 0;

  constructor(private maxConcurrent: number = 5) {}

  enqueue(task: QueuedTask): void {
    const key = task.groupId ?? task.userId;
    const queue = this.queues.get(key) ?? [];
    queue.push(task);
    this.queues.set(key, queue);
    this.processNext();
  }

  private async processNext(): Promise<void> {
    if (this.activeCount >= this.maxConcurrent) return;

    // 找到第一个有待处理任务的队列
    for (const [key, queue] of this.queues) {
      if (queue.length === 0) continue;

      // 同一群组串行执行（防止工具冲突）
      if (this.isGroupActive(key)) continue;

      const task = queue.shift()!;
      this.activeCount++;
      this.setGroupActive(key, true);

      // 异步执行，不阻塞队列处理
      this.executeTask(task)
        .catch(err => logger.error({ err, taskId: task.id }, 'Task failed'))
        .finally(() => {
          this.activeCount--;
          this.setGroupActive(key, false);
          this.processNext();  // 继续处理下一个
        });

      break;
    }
  }
}
```

**设计决策**：

- **同一群组/用户内串行**：防止两个并行容器操作同一工作目录产生冲突
- **不同群组间并行**：最多 `maxConcurrent` 个容器同时运行
- **背压处理**：当所有槽位占满时，新任务等待在内存队列中

### 1.2.6 任务调度器 (`src/task-scheduler.ts`)

**职责**：Cron 定时任务调度 + 延迟队列（解决 NanoClaw #830）。

```typescript
export class TaskScheduler {
  private deferredTasks = new Map<string, DeferredTask[]>();  // groupId → tasks
  private checkInterval: NodeJS.Timer | null = null;

  constructor(
    private db: Database,
    private groupQueue: GroupQueue,
  ) {}

  start(): void {
    // 每 30 秒检查一次定时任务
    this.checkInterval = setInterval(() => this.checkScheduledTasks(), 30_000);
  }

  private checkScheduledTasks(): void {
    const now = new Date();
    const dueTasks = this.db.getDueScheduledTasks(now);

    for (const task of dueTasks) {
      const groupKey = task.groupId ?? task.userId;

      // ★ 关键差异：不直接丢弃，而是检查是否可执行
      if (this.groupQueue.isGroupActive(groupKey)) {
        // 群组忙碌 → 入延迟队列（而非丢弃）
        this.deferTask(task);
      } else {
        this.groupQueue.enqueue(this.toQueuedTask(task));
        this.db.updateScheduledTaskLastRun(task.id, now);
      }
    }

    // 检查延迟队列中可以执行的任务
    this.processDeferredTasks();
  }

  private deferTask(task: ScheduledTask): void {
    const key = task.groupId ?? task.userId;
    const deferred = this.deferredTasks.get(key) ?? [];

    // 最大延迟 1 小时
    const maxDeferUntil = new Date(Date.now() + 3600_000);
    deferred.push({
      task,
      deferredAt: new Date(),
      maxDeferUntil,
      retryCount: 0,
    });

    this.deferredTasks.set(key, deferred);
    logger.warn({ taskId: task.id, groupKey: key }, 'Task deferred (group busy)');
  }

  private processDeferredTasks(): void {
    for (const [key, deferred] of this.deferredTasks) {
      if (this.groupQueue.isGroupActive(key)) continue;  // 仍然忙碌

      const now = new Date();
      const remaining: DeferredTask[] = [];

      for (const item of deferred) {
        if (now > item.maxDeferUntil) {
          // 超过最大延迟时间 → 记录警告并丢弃
          logger.warn({ taskId: item.task.id }, 'Deferred task expired (>1h)');
          continue;
        }

        // 执行延迟任务
        this.groupQueue.enqueue(this.toQueuedTask(item.task));
        this.db.updateScheduledTaskLastRun(item.task.id, now);
        break;  // 每次只执行一个（保持串行）
      }

      this.deferredTasks.set(key, remaining);
    }
  }
}
```

```typescript
import Docker from 'dockerode';

export class ContainerRunner {
  private docker: Docker;

  constructor(
    private config: DockerConfig,
    private credentialProxy: CredentialProxy,
  ) {
    this.docker = new Docker({ socketPath: config.socketPath });
  }

  /**
   * 执行一个 Agent 任务
   */
  async runTask(task: QueuedTask): Promise<TaskResult> {
    const containerId = `vigilclaw-${task.id}`;
    const ipcDir = this.prepareIpcDir(task.id);

    // 1. 写入 IPC 输入文件
    this.writeTaskInput(ipcDir, task);

    // 2. 创建并启动容器
    const container = await this.createContainer(containerId, ipcDir, task);
    await container.start();

    try {
      // 3. 监控输出（带超时）
      const result = await this.waitForResult(container, ipcDir, task.id);
      return result;
    } catch (err) {
      // 超时或异常 → 强制停止
      await this.forceStop(container);
      throw err;
    } finally {
      // 4. 清理容器和 IPC 文件
      await this.cleanup(container, ipcDir);
    }
  }

  private async createContainer(
    name: string,
    ipcDir: string,
    task: QueuedTask,
  ): Promise<Docker.Container> {
    return this.docker.createContainer({
      name,
      Image: this.config.image,
      Cmd: ['node', '/app/agent-runner.js'],
      Env: [
        // 不传 API Key！只传代理地址
        `CREDENTIAL_PROXY_URL=http://host.docker.internal:${this.credentialProxy.port}`,
        `TASK_ID=${task.id}`,
      ],
      HostConfig: {
        AutoRemove: true,
        ReadonlyRootfs: true,
        NetworkMode: 'vigilclaw-network',  // 受限网络

        // ---- 安全配置 ----
        SecurityOpt: ['no-new-privileges:true'],
        CapDrop: ['ALL'],                  // 移除所有 Linux capabilities

        // ---- 资源限制 ----
        Memory: parseMemory(this.config.memoryLimit),
        CpuQuota: parseCpu(this.config.cpuLimit),
        PidsLimit: this.config.pidsLimit,

        // ---- 临时可写层 ----
        Tmpfs: {
          '/tmp': 'rw,noexec,nosuid,size=100m',
        },

        // ---- 卷挂载 ----
        Binds: [
          `${ipcDir}:/ipc:rw`,                          // IPC 通道
          `${task.workspaceDir}:/workspace:rw`,          // 工作目录
        ],
      },
    });
  }

  /**
   * 等待 IPC 输出文件出现，或超时
   */
  private async waitForResult(
    container: Docker.Container,
    ipcDir: string,
    taskId: string,
  ): Promise<TaskResult> {
    return new Promise((resolve, reject) => {
      const timeout = setTimeout(() => {
        reject(new Error(`Task ${taskId} timed out`));
      }, this.config.taskTimeout);

      // 监控 IPC 输出目录
      const watcher = fs.watch(path.join(ipcDir, 'output'), (event, filename) => {
        if (filename === `result-${taskId}.json`) {
          clearTimeout(timeout);
          watcher.close();

          const resultPath = path.join(ipcDir, 'output', filename);
          const result = JSON.parse(fs.readFileSync(resultPath, 'utf-8'));
          resolve(result);
        }
      });

      // 同时监控容器退出（异常退出）
      container.wait().then(({ StatusCode }) => {
        if (StatusCode !== 0) {
          clearTimeout(timeout);
          watcher.close();
          reject(new Error(`Container exited with code ${StatusCode}`));
        }
      });
    });
  }

  /**
   * 优雅关闭所有运行中容器
   */
  async drainAll(timeoutMs: number): Promise<void> {
    const containers = await this.docker.listContainers({
      filters: { name: ['vigilclaw-'] },
    });

    await Promise.allSettled(
      containers.map(c =>
        this.docker.getContainer(c.Id).stop({ t: Math.floor(timeoutMs / 1000) })
      )
    );
  }
}
```

#### ReAct 循环实现

```typescript
import Anthropic from '@anthropic-ai/sdk';

const MAX_TURNS = 30;           // 安全阀：最多 30 轮工具调用
const SYSTEM_PROMPT = `You are a helpful AI assistant. You can use tools to help users.
When using the bash tool, always be careful and explain what you're doing.
If a command fails, analyze the error and try a different approach.`;

async function reactLoop(
  client: Anthropic,
  taskInput: TaskInput,
  tools: ToolDefinition[],
): Promise<TaskResult> {
  const messages: Anthropic.MessageParam[] = taskInput.messages;
  let totalInputTokens = 0;
  let totalOutputTokens = 0;

  for (let turn = 0; turn < MAX_TURNS; turn++) {
    // 检查是否有新注入的消息（Piping）
    const injected = checkInjectedMessages();
    if (injected.length > 0) {
      messages.push(...injected);
    }

    // 调用 Claude API
    const response = await client.messages.create({
      model: taskInput.model,
      max_tokens: taskInput.maxTokens,
      system: SYSTEM_PROMPT,
      messages,
      tools: tools.map(t => t.definition),
    });

    // 累计 token
    totalInputTokens += response.usage.input_tokens;
    totalOutputTokens += response.usage.output_tokens;

    // 将 assistant 消息加入历史
    messages.push({ role: 'assistant', content: response.content });

    // 写入流式输出（供宿主机实时转发）
    writeStreamOutput(response);

    // 模型返回文本，循环结束
    if (response.stop_reason === 'end_turn' || response.stop_reason === 'stop_sequence') {
      const textContent = response.content
        .filter((b): b is Anthropic.TextBlock => b.type === 'text')
        .map(b => b.text)
        .join('');

      return {
        taskId: taskInput.taskId,
        success: true,
        response: {
          content: textContent,
          usage: { inputTokens: totalInputTokens, outputTokens: totalOutputTokens },
          model: taskInput.model,
        },
      };
    }

    // 模型请求工具调用
    if (response.stop_reason === 'tool_use') {
      const toolResults: Anthropic.ToolResultBlockParam[] = [];

      for (const block of response.content) {
        if (block.type !== 'tool_use') continue;

        const tool = tools.find(t => t.definition.name === block.name);
        if (!tool) {
          toolResults.push({
            type: 'tool_result',
            tool_use_id: block.id,
            content: `Error: Unknown tool "${block.name}"`,
            is_error: true,
          });
          continue;
        }

        try {
          const output = await tool.execute(block.input);
          toolResults.push({
            type: 'tool_result',
            tool_use_id: block.id,
            content: truncateOutput(output, 50_000),   // 限制输出大小
          });
        } catch (err) {
          toolResults.push({
            type: 'tool_result',
            tool_use_id: block.id,
            content: `Error: ${(err as Error).message}`,
            is_error: true,
          });
        }
      }

      messages.push({ role: 'user', content: toolResults });
    }
  }

  // 超过最大轮次
  return {
    taskId: taskInput.taskId,
    success: false,
    response: {
      content: 'Agent reached maximum number of tool call turns.',
      usage: { inputTokens: totalInputTokens, outputTokens: totalOutputTokens },
      model: taskInput.model,
    },
  };
}
```

### 1.4.2 Claude Provider 实现 (`src/provider/claude.ts`)

```typescript
import Anthropic from '@anthropic-ai/sdk';

export class ClaudeProvider implements IProvider {
  readonly name = 'claude';
  private client: Anthropic;

  // 价格表（$/1M tokens）
  private static PRICING: Record<string, { input: number; output: number }> = {
    'claude-sonnet-4-5-20250929': { input: 3, output: 15 },
    'claude-haiku-3-5-20250929':  { input: 1, output: 5 },
    'claude-opus-4-20250929':     { input: 15, output: 75 },
  };

  constructor(baseURL?: string) {
    this.client = new Anthropic({
      // 容器内：通过 Credential Proxy 代理（baseURL = proxy 地址）
      // 宿主机：直接使用 ANTHROPIC_API_KEY 环境变量
      ...(baseURL ? { baseURL } : {}),
    });
  }

  async chat(params: ChatParams): Promise<ChatResponse> {
    const response = await this.client.messages.create({
      model: params.model,
      max_tokens: params.maxTokens ?? 4096,
      system: params.system,
      messages: this.toAnthropicMessages(params.messages),
      tools: params.tools as Anthropic.Tool[],
      temperature: params.temperature,
    });

    return {
      content: response.content,
      stopReason: response.stop_reason as ChatResponse['stopReason'],
      usage: {
        inputTokens: response.usage.input_tokens,
        outputTokens: response.usage.output_tokens,
      },
      model: response.model,
    };
  }

  async *stream(params: ChatParams): AsyncGenerator<ChatChunk> {
    const stream = this.client.messages.stream({
      model: params.model,
      max_tokens: params.maxTokens ?? 4096,
      system: params.system,
      messages: this.toAnthropicMessages(params.messages),
      tools: params.tools as Anthropic.Tool[],
    });

    for await (const event of stream) {
      if (event.type === 'content_block_start') {
        if (event.content_block.type === 'tool_use') {
          yield {
            type: 'tool_use_start',
            toolName: event.content_block.name,
            toolId: event.content_block.id,
          };
        }
      } else if (event.type === 'content_block_delta') {
        if (event.delta.type === 'text_delta') {
          yield { type: 'text_delta', text: event.delta.text };
        } else if (event.delta.type === 'input_json_delta') {
          yield { type: 'tool_input_delta', partialJson: event.delta.partial_json };
        }
      } else if (event.type === 'content_block_stop') {
        yield { type: 'block_stop' };
      } else if (event.type === 'message_stop') {
        yield { type: 'message_stop' };
      }
    }
  }

  toolDefinitions(tools: ITool[]): ToolDefinition[] {
    return tools.map(tool => ({
      name: tool.name,
      description: tool.description,
      input_schema: tool.schema,
    }));
  }

  estimateCost(inputTokens: number, outputTokens: number, model: string): number {
    const pricing = ClaudeProvider.PRICING[model];
    if (!pricing) return 0;
    return (inputTokens / 1_000_000) * pricing.input
         + (outputTokens / 1_000_000) * pricing.output;
  }

  private toAnthropicMessages(messages: Message[]): Anthropic.MessageParam[] {
    return messages
      .filter(m => m.role !== 'system')   // system 消息单独传
      .map(m => ({
        role: m.role as 'user' | 'assistant',
        content: m.content,
      }));
  }
}
```

```typescript
export class BashTool implements ITool {
  readonly name = 'bash';
  readonly description = 'Execute a shell command and return its output. Use for running programs, checking file status, installing packages, etc.';
  readonly schema = {
    type: 'object' as const,
    properties: {
      command: { type: 'string', description: 'The shell command to execute' },
    },
    required: ['command'],
  };

  async execute(params: Record<string, unknown>): Promise<string> {
    const command = params.command as string;
    const timeout = 120_000;   // 2 分钟超时

    try {
      const { stdout, stderr } = await execAsync(command, {
        timeout,
        maxBuffer: 1024 * 1024,   // 1MB 输出限制
        cwd: '/workspace',
      });

      const output = (stdout + (stderr ? `\nSTDERR:\n${stderr}` : '')).trim();
      return truncate(output, 50_000);   // 截断过大输出
    } catch (err: any) {
      if (err.killed) return `Error: Command timed out after ${timeout / 1000}s`;
      return `Error (exit code ${err.code}): ${err.stderr || err.message}`;
    }
  }
}
```

```typescript
      file_path: { type: 'string', description: 'Absolute path to the file to read' },
      offset: { type: 'number', description: 'Line number to start from (1-indexed, default: 1)' },
      limit: { type: 'number', description: 'Maximum number of lines to read (default: 2000)' },
    },
    required: ['file_path'],
  };

  async execute(params: Record<string, unknown>): Promise<string> {
    const filePath = params.file_path as string;
    const offset = (params.offset as number) ?? 1;
    const limit = (params.limit as number) ?? 2000;

    // 安全检查：仅允许 /workspace 下的文件
    if (!filePath.startsWith('/workspace')) {
      return 'Error: Can only read files under /workspace';
    }

    try {
      const content = fs.readFileSync(filePath, 'utf-8');
      const lines = content.split('\n');
      const selected = lines.slice(offset - 1, offset - 1 + limit);
      return selected.map((line, i) => `${offset + i}: ${line}`).join('\n');
    } catch (err: any) {
      return `Error: ${err.message}`;
    }
  }
}
```

```typescript
export class EditTool implements ITool {
  readonly name = 'edit';
  readonly description = 'Make a precise edit to a file by replacing an exact string match with new content.';
  readonly schema = {
    type: 'object' as const,
    properties: {
      file_path: { type: 'string', description: 'Absolute path to the file' },
      old_string: { type: 'string', description: 'Exact string to find and replace' },
      new_string: { type: 'string', description: 'Replacement string' },
    },
    required: ['file_path', 'old_string', 'new_string'],
  };

  async execute(params: Record<string, unknown>): Promise<string> {
    const filePath = params.file_path as string;
    const oldString = params.old_string as string;
    const newString = params.new_string as string;

    if (!filePath.startsWith('/workspace')) {
      return 'Error: Can only edit files under /workspace';
    }

    try {
      const content = fs.readFileSync(filePath, 'utf-8');

      const occurrences = content.split(oldString).length - 1;
      if (occurrences === 0) {
        return 'Error: old_string not found in file';
      }
      if (occurrences > 1) {
        return `Error: Found ${occurrences} matches. Provide more context to identify a unique match.`;
      }

      const newContent = content.replace(oldString, newString);
      fs.writeFileSync(filePath, newContent, 'utf-8');
      return `Successfully edited ${filePath}`;
    } catch (err: any) {
      return `Error: ${err.message}`;
    }
  }
}
```

### 1.5.3 工具安全约束

所有工具的安全保障由**容器隔离**提供，工具代码本身不需要（也不应该）实现复杂的安全逻辑：

| 安全层 | 由谁保障 | 工具层面做什么 |
|-------|---------|-------------|
| 文件系统隔离 | Docker 容器 | 仅检查路径前缀（`/workspace`）作为 defense-in-depth |
| 网络隔离 | iptables + Docker network | 工具无需关心 |
| 资源限制 | Docker `--memory`, `--cpus` | 工具超时交给 `execAsync` |
| 凭证保护 | Credential Proxy | 工具完全看不到 API Key |
